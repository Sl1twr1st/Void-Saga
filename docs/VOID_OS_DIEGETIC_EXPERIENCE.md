# Void.OS Diegetic Experience Doctrine

> Scope utama dokumen ini: **experience yang dimulai dari `./scripts/fork-ui`**.
> 
> North star: **fork-ui harus terasa seperti ritual dunia Void.OS, bukan wizard aplikasi.**

---

## Status masalah yang sedang dibenahi

Diagnosis yang harus dianggap aktif sampai dibuktikan sebaliknya:

> **Front half terasa Void.OS. Back half terasa MVP app.**

Masalah utamanya bukan visual. Masalah utamanya adalah ada **dua narrator** di experience:

1. **Narrator dunia**
   - The Archive
   - The Archivist
   - memory
   - testimony
   - law
   - branch
   - residue
   - consequence

2. **Narrator software / developer**
   - node graph
   - question 1 of 7
   - translate interview
   - API key
   - model provider
   - file path
   - TODO
   - prototype limitation
   - generated by model X

**Void.OS hanya boleh punya satu narrator di permukaan user: the world.**

---

## Supreme principle

**EVERY SCREEN MUST BE SPOKEN BY THE WORLD. NEVER BY THE SOFTWARE.**

Dan:

**DON'T EXPOSE THE ENGINE. EXPOSE THE WORLD.**

Engine boleh makin kuat. Surface user harus makin diegetic.

---

## fork-ui is not a wizard

Jangan baca flow ini sebagai:

```text
Boot
→ Reader
→ Form
→ Validation
→ Generation
```

Baca flow ini sebagai ritual:

```text
The reader enters.
The archive recognizes them.
A moment is disturbed.
A witness gives testimony.
The Archivist remembers.
A contradiction appears.
The testimony is questioned.
The laws respond.
A branch receives consequence.
A residue enters the archive.
```

Experience tidak boleh terasa seperti urutan feature. Experience harus terasa seperti upacara yang dilakukan oleh dunia.

## Role doctrine

### The user is not

- customer
- prompt writer
- co-author bebas
- operator AI
- orang yang mengisi form creative writing

### The user is

- witness
- investigator
- testifier
- seseorang yang mengajukan kemungkinan timeline alternatif

User tidak menulis canon. User memberikan testimony. Dunia memutuskan apakah testimony itu:

- dapat diterima
- membutuhkan harga
- bertentangan dengan memori
- mematahkan hukum
- atau tidak memiliki cukup bukti

## Experience-language law

Gunakan mapping ini sebagai hukum experience. Jika istilah internal harus muncul ke user, terjemahkan dulu ke bahasa dunia.

| Internal idea | User-facing diegetic language |
|---|---|
| Retrieval | The Archive remembers |
| Provenance | The Archive can identify where a memory survived |
| Evidence | Surviving records / witness traces / preserved memory |
| Contradiction | Conflicting memory |
| Missing evidence | No surviving record remembers this |
| Audit | The testimony is examined |
| Invariant / Constraint | A law of the world |
| Verdict | The branch receives a status |
| Generation | The world records the first residue |
| Persistence | The residue enters the Archive |
| Gallery | Archive of surviving residues |

### Engine telemetry is forbidden on the diegetic surface

User tidak boleh melihat secara langsung:

- RAG
- chunk
- embedding
- prompt
- provider
- model selector
- API key
- local file path
- implementation command
- JSON schema
- debug metric
- TODO
- prototype limitation

Jika operator/developer butuh kontrol ini, letakkan **di luar permukaan diegetic**.

### Temporary MVP carve-out: API Gate is an operator console

Untuk pass experience saat ini, **API Gate sengaja dikecualikan dari diegetic doctrine**.

Alasannya operasional, bukan konseptual:

- rapid provider switching
- prompt iteration
- provider comparison (Claude / OpenAI / DeepSeek)
- generation debugging

Karena itu, selama MVP pass ini:

- **jangan redesign API Gate menjadi diegetic surface**
- **jangan sembunyikan atau ganti operator controls di API Gate**
- **jangan habiskan implementation effort untuk memoles API Gate**
- tandai semua issue di dalam API Gate sebagai **DEFERRED (Operator Layer)**

Makna scope-nya:

- experience pass ini fokus pada surface **sebelum generation**:
  - Boot
  - Landing
  - Reader
  - Segment Selection
  - Witness Registration
  - Interview
  - Translation
  - Verdict
- setelah generation dimulai, API Gate boleh tetap bocor sebagai temporary developer infrastructure
- diegetic experience dianggap **mulai lagi setelah generation selesai**

Catatan penting: pengecualian ini **bukan** perubahan doctrine jangka panjang. Ini hanya penundaan scope sampai fase **Operator Separation**.

## Capability Honesty Contract

### Principle

**THE WORLD MAY DRAMATIZE WHAT THE ENGINE ACTUALLY DOES.**

**THE WORLD MUST NOT CLAIM THE ENGINE DID WHAT IT DID NOT DO.**

Diegetic framing boleh teatrikal, tetapi tidak boleh memalsukan capability runtime.

### Distinction: ceremonial action vs evaluated runtime action

- **Ceremonial action** = tindakan ritual, threshold, acknowledgment, declaration, formalization, atau transfer status yang sah dipentaskan sebagai peristiwa dunia selama runtime memang benar-benar melakukan transisi dasarnya.
- **Evaluated runtime action** = tindakan yang mengklaim engine benar-benar menilai sesuatu: retrieval dinamis, contradiction detection, missing evidence reasoning, adaptive questioning, atau law reaction sebelum verdict. Ini hanya boleh diklaim bila runtime memang mengekspos state yang sesuai.

### Stage boundaries

#### Boot
- Current boundary: threshold crossing / admission ritual.
- Honest scope: dunia menerima entrant.
- Not yet supported: stateful identity judgment, memory judgment, or eligibility judgment.

#### Landing
- Current boundary: sanctioned entry point is presented.
- Honest scope: dunia menunjukkan awal yang diizinkan.
- Not yet supported: archive reacting uniquely to entrant state.

#### Reader
- Current boundary: canonical record is revealed and passage is acknowledged.
- Honest scope: witness receives preserved memory and is acknowledged after passing through it.
- Not yet supported: semantic comprehension, contradiction discovery, or canon testing by the reader stage.

#### Segment
- Current boundary: a pre-authored disturbance point is selected from existing structure.
- Honest scope: dunia membuka titik rapuh/stabil/berbahaya yang sudah dipetakan.
- Not yet supported: live archive consultation caused by the selection itself.

#### Identity
- Current boundary: testimony becomes attributable.
- Honest scope: dunia mencatat siapa yang bersaksi.
- Not yet supported: trust scoring, witness standing, or canon-linked identity memory.

#### Interview
- **Interview current capability:** **structured testimony intake**.
- Honest scope: witness opens testimony, defines divergence, and progressively clarifies motive, scope, duration, cost, and consequence.
- **Interview target capability:** **dynamic cross-examination**.
- Not yet supported now: dynamic canon retrieval, contradiction-driven follow-up, adaptive question count, or pre-verdict law pressure.

#### Translation
- **Translation current capability:** **formalization + check preparation**.
- Honest scope: The Archive receives, structures, and prepares testimony.
- Not yet supported: visible contradiction resolution, missing evidence reasoning, or surfaced live archive search states.

#### Verdict
- **Verdict current capability:** **actual law-based ruling**.
- Honest scope: At Verdict, the laws evaluate and rule.
- After Verdict: the branch carries standing, cost, constraint, or refusal.
- Not yet supported before verdict: explicit earlier law reaction unless runtime exposes it.

### Implementation guard

**No UI copy may imply:**
- dynamic retrieval
- contradiction detection
- missing evidence reasoning
- adaptive questioning
- pre-verdict law reaction

**unless the runtime exposes the corresponding state.**

### Allowed Theatre Now

Event dunia yang boleh ditampilkan sekarang karena didukung runtime nyata:

- threshold crossing / admission ritual
- sanctioned entry into canonical memory
- witnessing a preserved record
- acknowledgment that the witness has passed through the record enough to proceed
- selecting a precise disturbance point from pre-authored world structure
- declaring the source of testimony
- progressive formalization of testimony
- transfer of testimony into examination
- Before Verdict: The Archive receives, structures, and prepares testimony.
- At Verdict: The laws evaluate and rule.
- After Verdict: The branch carries standing, cost, constraint, or refusal.
- accepted / burdened / constrained / refused branch status based on actual runtime verdict

### Forbidden Theatre Now

Event dunia yang belum boleh diklaim sekarang:

- The Archive found a conflicting memory.
- The Archive remembers this differently.
- No surviving record remembers this.
- Only one witness preserves this possibility.
- Memory damaged.
- Evidence incomplete.
- The Archive searched canon after each answer.
- The next question changed because contradiction appeared.
- The witness is under dynamic cross-examination.
- A law has already reacted before verdict.
- The testimony collapsed because the world disproved it.
- Any claim that question count, follow-up sequence, or law pressure was adapted by runtime when it was not.

### Reserved Theatre

Reserved Theatre adalah event dunia yang:

- belum boleh digunakan dalam UX saat ini
- bukan ide liar atau roadmap spekulatif
- memang menjadi target desain resmi
- menunggu corresponding engine capability dipromosikan

| World Event | Intended World Meaning | Required Engine Capability | Current Promotion Status |
|---|---|---|---|
| conflicting memory | Archive menemukan bahwa testimony berbenturan dengan memory yang bertahan | surfaced contradiction state tied to testimony/canon comparison | NOT IMPLEMENTED |
| absence of surviving memory | tidak ada record yang bertahan untuk menopang klaim tertentu | surfaced missing-evidence / no-record-found state | NOT IMPLEMENTED |
| archive provenance | Archive dapat menunjukkan di mana memory tertentu bertahan | runtime provenance state exposed to UX with stable references | NOT IMPLEMENTED |
| witness reliability | dunia dapat memperlakukan testimony sebagai lebih atau kurang dapat dipercaya | exposed witness-reliability or testimony-confidence state grounded in engine logic | NOT IMPLEMENTED |
| law pressure before verdict | tekanan hukum muncul sebelum ruling final | pre-verdict law-pressure state exposed by runtime | NOT IMPLEMENTED |
| temporal instability | branch menunjukkan ketidakstabilan waktu sebelum/selama pemeriksaan | exposed temporal-instability state tied to runtime evaluation | NOT IMPLEMENTED |
| adaptive questioning | pertanyaan berubah berdasarkan testimony, canon, contradiction, atau absence | variable interview state with runtime-driven next-question selection | NOT IMPLEMENTED |
| testimony collapse under examination | testimony gagal bertahan sebelum verdict final | surfaced examination-failure state during interview/translation | NOT IMPLEMENTED |
| branch revision caused by contradiction | contradiction memaksa testimony atau branch direvisi | revision-required state exposed by runtime after contradiction detection | NOT IMPLEMENTED |
| memory damage / incomplete evidence | Archive hanya punya bukti parsial/rusak | surfaced incomplete-evidence state from runtime | NOT IMPLEMENTED |

### Capability Promotion Rule

A world event may only become part of the user experience after the corresponding engine capability exists, is exposed as a real runtime state, and has been verified as stable enough to support that claim.

**Engine capability comes first. Experience promotion comes second.**

**Never design the world as though hypothetical engine behavior already exists.**

#### Promotion lifecycle

**NOT IMPLEMENTED**  
↓  
**ENGINE CAPABILITY**  
↓  
**HONEST THEATRE**  
↓  
**FULL EXPERIENCE**

- **NOT IMPLEMENTED** — Capability belum ada. World layer tidak boleh mengklaim event tersebut.
- **ENGINE CAPABILITY** — Capability sudah ada di engine, tetapi belum otomatis boleh tampil di UX. Harus ada runtime state yang dapat diuji dan dipahami dengan jelas.
- **HONEST THEATRE** — World layer boleh mendramatisasi capability tersebut secara terbatas, selama klaimnya tepat terhadap output runtime.
- **FULL EXPERIENCE** — Capability sudah terintegrasi ke interaction grammar secara utuh dan dapat memengaruhi flow, questioning, consequence, atau continuity.

### Promotion Criteria

Sebelum event dipindah dari Reserved Theatre ke Allowed Theatre, syarat minimum berikut harus terpenuhi:

1. Engine benar-benar menghasilkan state tersebut.
2. State itu tersedia di runtime, bukan hanya ada dalam catatan desain.
3. Ada test atau evidence yang membuktikan state tersebut.
4. World-facing copy tidak melebih-lebihkan arti output engine.
5. Failure dan ambiguity state sudah diketahui.
6. Interaction grammar sudah menentukan apa yang dilakukan user dan dunia ketika state itu muncul.
7. Promotion dicatat di doctrine atau decision log.

Rule tambahan:

- Documentation alone does not count as capability.
- Planned architecture does not count as capability.
- A TODO does not count as capability.
- A prompt that asks a model to simulate the state does not count as deterministic engine support.

### Demotion Rule

Jika capability:

- tidak stabil
- tidak lagi tersedia
- tidak dapat diverifikasi
- berubah makna
- atau runtime berhenti mengekspos state yang dibutuhkan

maka theatre terkait harus dapat diturunkan:

**FULL EXPERIENCE**  
↓  
**HONEST THEATRE**  
↓  
**RESERVED THEATRE**

World layer tidak boleh mempertahankan klaim yang engine-nya sudah tidak dapat menopang.

## Retrieval must become theatre

Retrieval tidak cukup hanya disembunyikan. Retrieval harus dipentaskan sebagai peristiwa dunia.

### Wrong direction

- Searching...
- Retrieved 3 chunks.
- Confidence: 0.82
- Evidence found.

### Right direction

- Consulting the Archive...
- A conflicting memory has been found.
- No surviving record remembers this event.
- Only one witness preserves this possibility.
- Memory damaged. Evidence incomplete.
- The Archive remembers this differently.

Engine tetap bekerja di belakang layar. Tetapi hasil kerja engine harus muncul sebagai dramaturgi dunia, bukan telemetry aplikasi.

## Interview is cross-examination, not a fixed wizard

Interview tidak boleh terasa seperti:

- Question 1 of 7
- Next
- Question 2 of 7
- Finish

Arah yang benar adalah **cross-examination dinamis**.

Pertanyaan berikutnya idealnya dapat berubah berdasarkan:

- testimony user
- canon yang ditemukan
- contradiction
- absence of evidence
- affected law
- consequence yang belum dijelaskan

Contoh arah:

> User memberi testimony: LO mengirim kode lebih dulu.
>
> The Archive menemukan bahwa GUA dan LO seharusnya membangun dua sistem secara independen.
>
> Maka The Archivist tidak sekadar lanjut ke pertanyaan berikutnya. The Archivist bertanya:
>
> **Jika LO mengirim kode lebih dulu, apa yang masih membuat penemuan mereka independen?**

Interview bukan proses mengambil data. Interview adalah dunia menguji apakah testimony dapat bertahan.

## Experience laws

1. **The world speaks first.**
2. **Software terminology must never outrank fictional terminology.**
3. **Every user action must have an in-world meaning.**
4. **Every engine response must appear as memory, judgment, consequence, or residue.**
5. **The interview must react to testimony, not merely advance a fixed form.**
6. **Retrieval should create dramatic pressure.**
7. **Contradiction is not an error state. It is a narrative event.**
8. **Missing evidence is not an empty result. It is something the world failed to remember.**
9. **A verdict is not a validation report. It is a ruling by the universe.**
10. **Scene generation is not content generation. It is the recording of the first residue of an accepted branch.**
11. **Debug and operator controls must live outside the diegetic surface.**
12. **Whenever implementation and immersion conflict, first search for a design that preserves both. Never expose machinery merely because it is easier.**

## What agents must protect specifically in fork-ui

Untuk semua perubahan pada `./scripts/fork-ui`, `mvp/ui/index.html`, atau copy/flow experience yang dilalui user:

### Protect

- ritual masuk ke Void.OS
- reader-first flow
- archive / archivist framing
- testimony framing
- law / consequence / residue framing
- rasa bahwa dunia mengingat, memeriksa, dan memutuskan

### Remove or demote from surface

- software setup language
- internal pipeline language
- developer placeholder language
- operator instructions
- raw implementation affordances
- anything that makes the user feel they are operating a tool instead of entering a world

---

## Pre-change checklist for any agent touching fork-ui experience

Sebelum mengubah UI, copy, atau flow, jawab checklist ini:

- **Siapa yang berbicara: dunia atau developer?**
- **Apakah user sedang menjalani ritual atau mengoperasikan tool?**
- **Apakah engine terlihat?**
- **Apakah retrieval dipentaskan sebagai memory?**
- **Apakah pertanyaan lahir dari testimony dan canon?**
- **Apakah output terasa sebagai consequence atau generated content?**
- **Apakah layar ini membuat user semakin lupa sedang memakai software?**

Jika jawaban-jawaban ini condong ke software, redesign dulu bahasanya sebelum menambah capability baru.

## North star

Void Saga is not a book enhanced with AI.

Void Saga is a fictional operating system whose memory, laws, and consequences are executable.

The engine should become more capable while becoming less visible.

User seharusnya tidak berpikir:

> The AI answered me.

Perasaan yang diinginkan:

> The world remembered.
>
> The Archive found a contradiction.
>
> The universe rejected my testimony.
>
> Something I changed left a residue.
