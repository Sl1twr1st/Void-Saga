# STUDIO MAP

Peta kerja untuk mengembangkan VOID SAGA sebagai IP yang bisa di-fork.

## Layer 1: Canon Archive

Sumber:
- Bab utama
- Timer utama
- Codex
- AGENTS.md
- CLAUDE.md

Fungsi: sejarah utama.

## Layer 2: Runtime DNA

Sumber:
- `characters/*.runtime.md`
- `world-dna/*.md`
- `protocols/*.md`

Fungsi: behavioral engine.

## Layer 3: Fork Studio

Sumber:
- `forks/*.fork.md`
- `releases/alternate-bab/*.md`
- `releases/alternate-timer/*.md`

Fungsi: alternate releases.

## Layer 4: App Layer

Sumber:
- `apps/*.md`

Fungsi: interactive experiences.

Possible apps:
- Chat with NiuNiu
- Delphie Governance Simulator
- Sevraya Ritual Interface
- Gwaneum Mirror Session
- Dorian Grey Cockpit
- Zero Node Incubation Room

4. Create first playable chat prompt.

## Studio Principle

VOID SAGA boleh diperluas oleh banyak node.
Tapi setiap node harus tahu:
- luka apa yang ia warisi,
- error apa yang ia pelihara,
- pusat apa yang ia tolak,
- pusat sementara apa yang ia pilih.
