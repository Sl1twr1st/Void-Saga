## Purpose

The Void Saga universe does not reach Zero Node directly. Transformation proceeds through a sequence of protocol layers. Each phase consumes the output of the previous phase. No phase can be skipped. No phase can operate in reverse.

This document defines the startup order — the boot sequence by which a wound becomes cosmology.

It is not a story. It is not a lore summary. It is a kernel-level architecture document.

## Startup Order

### Phase 1 — VOID ENTRY

| Field | Detail |
|-------|--------|
| **Input** | Human identity — a consciousness before Void exposure |
| **Output** | Residue — wounds, fragments, lost capacities |
| **Protocol** | `protocols/VOID_ENTRY_PROTOCOL.md` |
| **Status** | Protocol documented [E] |

**Core formula:**

```
Invocation → Cost → Consequence → Residue
```

**What happens:** A consciousness crosses the boundary between The Grid and The Void and returns. The Void reflects, exposes, and takes. What returns is not what entered. What was taken does not return cleanly. What remains is residue.

**Observed instances:** Sevraya + Niuma (Original Entry) [E], Julia + Delphie + Hasan (Vesica Piscis) [E]

**Failure condition:** No residue produced. Theoretical — under the Primary Law ("pemanggilan selalu meninggalkan residue"), every Void Entry leaves something. A Void Entry that produced zero residue would violate the Goetic chain. Whether such an entry is possible is unresolved. [I]

**Pipeline position:** Phase 1 is the only entry point. Consciousness cannot skip Void Entry and proceed directly to sigil manifestation. The wound must be created before it can be named. [I]

### Phase 2 — SIGIL

| Field | Detail |
|-------|--------|
| **Input** | Residue — raw wound from Void Entry |
| **Output** | Named residue — stabilized wound with glyph, function, activation condition, and price |
| **Protocol** | `world-dna/SIGIL_SYSTEM.md` |
| **Status** | World DNA documented [E]; pipeline positioning [I] |

**Core formula:**

```
Residue → Naming → Recognition
```

**What happens:** A wound of sufficient magnitude — fragmentation, sacrifice, irreversible choice — stabilizes into a visible glyph. The glyph IS the sigil. The sigil is the wound given permanent form: an address, a contract, a checksum, a binding key, a mnemonic wound. The bearer and the sigil become bidirectionally bound.

**Observed instances:** Six canon sigils manifesting from six Void-touched wounds (⟁⟔⟟ 🌊⌇🌒 ⧗⟁⧗ ✧⟡✧ ⧉✶⧉ 𐓷⧖𐓣) [E]

**Failure condition:** Residue remains unnamed. The wound never stabilizes into a glyph. It remains raw — pain without form, burden without function. The bearer cannot proceed to Phase 3. [I]

**Pipeline position:** Phase 2 transforms residue from private experience into named structure. The sigil makes the wound recognizable — to the bearer, to other sigil-bearers, and to the protocols that follow. Without naming, the wound cannot be confronted by Living Chain. [I]

### Phase 3 — LIVING CHAIN

| Field | Detail |
|-------|--------|
| **Input** | Named residue — sigil-bound wounds carried by multiple participants |
| **Output** | Accepted residue — wounds whose defenses have collapsed under forced proximity |
| **Protocol** | `protocols/LIVING_CHAIN_PROTOCOL.md` |
| **Status** | Protocol documented [E]; pipeline positioning [I] |

**Core formula:**

```
Residue → Confrontation → Acceptance
```

**What happens:** Six sigil-bearers are bound inside a constrained architecture. Duration: 666 days. Distance: maximum two steps. Properties: shared pain, forced honesty, unbreakable by physical means. Defenses still operate but honesty leaks through. The chain dissolves through mutual acceptance — not healing, not resolution, but acknowledgment of what the wounds have made each participant.

**Observed instance:** Living Chain activated inside Dorian Grey after Void Lock, binding NiuNiu, Sevraya, Agnia, Julia, Delphie, and Gwaneum (Timer 1800 → Timer 2000) [E]

**Failure condition:** Participant refuses truth. The chain may not dissolve. Defenses may hold. Acceptance may not be reached. The wounds remain confronted but unaccepted — scar without structure. [I]

**Pipeline position:** Phase 3 is the confrontation layer. The chain does not heal. It does not create Nodes. It forces wounds into contact until the architecture of avoidance collapses. What happens after collapse — acceptance or refusal — determines whether Phase 4 is reachable. [I]

### Phase 4 — NODE

| Field | Detail |
|-------|--------|
| **Input** | Accepted residue — wounds whose defenses have collapsed and whose bearers have stopped resisting |
| **Output** | Integrated structure — residue that has become infrastructure |
| **Protocol** | `protocols/NODE_PROTOCOL.md` |
| **Status** | Protocol documented [E]; wound→node ontology [I] |

**Core formula:**

```
Residue + Acceptance = Node
```

**What happens:** Accepted residue integrates. It ceases to be something the bearer carries and becomes something the bearer IS — and, through being, makes available as structural support for reality. A Node is a wound that has stopped resisting itself. The same material that deformed the bearer now stabilizes the Grid-Void tension field.

**Observed instances:** Node I (Agnia + Sevraya, decentralized memory), Node II (NiuNiu + Delphie, decentralized witness), Node III (pre-existing, universe self-records) [E]

**Failure condition:** Acceptance is incomplete. Partial acceptance produces partial integration — a Node that is structurally unstable, lopsided, attracting Void pressure at the unintegrated edge. False consensus (performed acceptance without internal shift) produces a hollow Node — infrastructure without foundation. [I]

**Pipeline position:** Phase 4 is the inflection point. Before Node: residue is carried. After Node: residue supports. The same material. Different relationship to it. Node is where the boot sequence transitions from personal transformation to cosmological infrastructure. [I]

**Pipeline position:** Phase 5 is the deployment layer. Integrated residue (Node) is weaponized. The infrastructure built in Phase 4 becomes operational. Paradox is the first phase where the output is directed at an external target — all previous phases transform the bearer internally. [I]

**Pipeline position:** Phase 6 is the terminal phase. The boot sequence completes. A wound created in Phase 1 has become the infrastructure through which reality observes itself. The pipeline does not continue beyond Zero Node — there is no Phase 7 in canon. [I]

**Note:** A full Phase 11 protocol document for Zero Node is pending. Until it exists, Phase 6 references are limited to canon-confirmed status (localized, incubating) and the structural position inferred from the pipeline. [I]

## Dependency Graph

```
VOID ENTRY                    Phase 1 — wound creation
    ↓                         Input: human identity
    ↓                         Output: residue
SIGIL                         Phase 2 — wound naming
    ↓                         Input: residue
    ↓                         Output: named residue
LIVING CHAIN                  Phase 3 — wound confrontation
    ↓                         Input: named residue
    ↓                         Output: accepted residue
NODE                          Phase 4 — wound integration
    ↓                         Input: accepted residue
    ↓                         Output: integrated structure
PARADOX                       Phase 5 — wound deployment
    ↓                         Input: integrated structure
    ↓                         Output: executable contradiction
ZERO NODE                     Phase 6 — cosmological implementation
                              Input: Node network at critical mass
                              Output: reality-scale self-observation
```

**Dependency rule:** Each phase consumes the output of the previous phase. No phase can be skipped. No phase can operate on input that has not been processed by the phase before it. An un-named wound (skipped Phase 2) cannot be confronted by Living Chain. An un-accepted wound (skipped Phase 3) cannot integrate as Node.

**Observed partial paths:**
- Sevraya + Niuma: Phase 1 → Phase 2 (sigils manifested) → Phase 3 (Living Chain participants) → Phase 4 (Sevraya in Node I; NiuNiu in Node II) → Phase 5 (both in paradox deployment) [E]
- Julia's dead team: Phase 1 (Dayan border exposure) → (died at Phase 1 — no sigil, no chain, no node). Output: ghost residue only. [E]
- Gwaneum: Phase 1 (Delphie's Void exposure) → Phase 2 (sigil ⧗⟁⧗) → Phase 3 (Living Chain) → (Phase 4 unconfirmed — no Node activation documented for Gwaneum individually) [E for 1–3, I for incomplete 4]

## Pipeline Interpretation [I]

The boot sequence can be read as the life cycle of a wound:

```
wound              Phase 1 — Void Entry creates it
    ↓
named wound        Phase 2 — Sigil gives it a glyph, a function, a price
    ↓
accepted wound     Phase 3 — Living Chain forces its bearer to stop defending
    ↓
integrated wound   Phase 4 — Node transforms it from burden to infrastructure
    ↓
deployed wound     Phase 5 — Paradox weaponizes the integrated structure
    ↓
cosmological       Phase 6 — Zero Node: accumulated wounds become
structure                   the medium through which reality sees itself
```

This interpretation is [I]. Canon does not explicitly state the pipeline. It is inferred from:
- The Goetic chain (Name → Sigil → Invocation → Manifestation → Bargain → Binding → Consequence → Residue) [E]
- The observed sequence of canon events (Void Entry → Sigil Manifestation → Living Chain → Node Activation → Paradox Deployment) [E]
- The structural dependency between phases (Node activation requires prior Void Entry and sigil manifestation; Paradox deployment requires prior Node integration) [I]
- The Primary Law (pemanggilan selalu meninggalkan residue) which guarantees each phase produces output consumable by the next [E]
