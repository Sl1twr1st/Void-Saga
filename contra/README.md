# contra/

> The adversarial record. Everything in this folder argues *against* what was built.

This folder exists to hold counterarguments, devil's-advocate readings, and brutal-honesty critiques of Void Saga / Executable Fiction — deliberately one-sided **against** the project.

It is not a balanced view. It is the prosecution. The rest of the repo is the defense.

## Why this exists

The project has a strong internal logic and a self-reinforcing vocabulary ("research program," "theory stabilization," "emergence detection"). That logic is persuasive — including to its author. A persuasive internal logic is exactly the condition under which blind spots survive longest.

`contra/` is the place where the project is allowed to be wrong, premature, or self-deceiving. Anything here should be read as: *"this is the strongest case someone could make against the current direction."* It is kept so the author can return to it when the internal story feels too clean.

## Rule

- Nothing in `contra/` modifies anything outside `contra/`. It only argues.
- Every entry is dated.
- Entries are not deleted when disproven — they are answered. A disproven contra is evidence the project survived a real challenge. Keep the trail.
